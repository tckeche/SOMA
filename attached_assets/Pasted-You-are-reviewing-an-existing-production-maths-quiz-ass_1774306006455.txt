C. PHASE 1 FEATURE VERIFICATION

1) GRAPH GENERATION
Verify:
- graph questions can be created/rendered
- graph display is readable and not cramped
- axes, labels, ranges, grid, and plotted data are visible
- graph rendering is deterministic and not broken
- non-graph questions still render normally
- mobile/tablet responsive layout is still acceptable if responsive UI exists

Test cases:
- straight-line graph
- simple curve graph
- plotted points graph
- edge case with negative and positive axes

2) SYLLABUS PDF UPLOAD + GROUNDING
Verify:
- syllabus upload UI works
- required metadata exists: board, level, syllabus code
- text-based PDF upload succeeds
- extracted content is stored/retrievable
- selected syllabus context is used during question generation
- invalid or non-extractable PDF fails clearly and safely
- app does not crash on bad upload
- existing upload flows elsewhere are unaffected

Test cases:
- valid text PDF
- invalid PDF
- empty upload or missing metadata
- selecting one syllabus among multiple uploaded syllabi

3) COPILOT SESSION MEMORY DURING ASSESSMENT BUILDING
Verify:
- conversation context persists within one active assessment-generation session
- tutor follow-up changes are remembered
- previewing does not accidentally clear memory
- exiting assessment builder does clear/end the session
- starting a new assessment does not incorrectly reuse old session memory
- progress/status UI shows meaningful stages instead of only a vague spinner

Test cases:
- initial request
- follow-up refinement
- preview
- return to editing
- exit builder
- start fresh new builder session

4) COPILOT SUMMARY OUTPUT
Verify:
- after generation or revision, copilot returns a visible summary
- summary includes:
  - number of questions
  - question types
  - topics
  - subtopics
  - difficulty or difficulty mix
  - syllabus grounding context if used
- summary matches actual generated output

5) ANSWER RANDOMISATION BALANCE
Verify:
- correct answers are not biased heavily toward A or B
- answer positions are distributed across A/B/C/D
- exact balance when question count divisible by 4, or near-even balance otherwise
- correctness is preserved after shuffling
- distractor order feels natural

Test cases:
- 4-question quiz
- 8-question quiz
- 10-question quiz
- repeated generation runs to inspect distribution behaviour

6) LIGHT MODE REMOVAL
Verify:
- light mode toggle is gone or disabled
- dark mode is forced consistently
- app visuals are not broken due to theme removal
- no dead toggle/button remains
- no page has unreadable text due to theme assumptions

7) BUTTON SIZE FIXES
Verify:
- Back, Exit Preview, and Exit Assessment buttons match normal button size
- padding, height, font size, and tap/click area are correct
- they visually align with the app’s standard buttons
